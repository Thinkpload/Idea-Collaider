# Коллайдер идей · Idea Collider

**Память как топливо для идей.** Не генерация с чистого листа — а *столкновение*.

Idea Collider — это [skill для Claude.ai](https://www.anthropic.com/news/skills), который порождает идеи не из воздуха, а через **столкновение далёких нитей из твоей собственной памяти и контекста**. Обычный брейншторм выдаёт то, что подошло бы кому угодно. Коллайдер берёт нити, которые накопил конкретно ты, и сталкивает самые далёкие друг от друга — новое рождается на стыках, которые ты сам бы не соединил.

> **Один принцип держит всё:** новизна = семантическая дистанция между столкнутыми нитями. Близкие нити дают банальность, чистый рандом — абсурд. Вся работа метода — найти далёкие, но соединимые пары и довести их до собираемого MVP.

🔗 **Лендинг / живое демо:** разворачивается на Vercel из [`index.html`](index.html) (ссылку добавим после деплоя — см. [PLAN.md](PLAN.md)).

## Как это работает

### Откуда берутся нити

Перед столкновением скилл собирает инвентарь нитей про **этого** пользователя из всего, к чему есть доступ в рантайме:

- **Память** о прошлых разговорах — проекты, стек, рынок, роль, повторяющиеся темы.
- **Подключённые источники**, если они есть (заметки встреч, документы, задачи, переписка) — только то, что реально доступно.
- **Личные и культурные нити** — хобби, кино, книги, игры, музыка. Они критичны: почти вся искра приходит отсюда, а не из рабочего стека.

### Онбординг происходит автоматически

Пользователь не заполняет анкету вручную. Контекст, важный для работы скилла, **собирается автоматически** из доступной памяти и подключённых источников в рантайме внутри Claude.ai. Инвентарь нитей показывается обратно пользователю для подтверждения и правки — это и портрет, и чистка данных. Если памяти почти нет (новый пользователь), скилл проводит короткий интейк: просит 5–8 нитей вперемешку рабочих и личных.

### Протокол — четыре шага

| Шаг | Что происходит |
|-----|----------------|
| **01 · Инжекция** | Память выгружает домены как частицы в камеру. 8–14 нитей с балансом каналов `работа` / `личное·культура`. |
| **02 · Разгон** | Сталкиваем самые далёкие пары. Главное правило: **поперёк каналов** — одна рабочая нить × одна личная. Это держит дистанцию в рабочем коридоре. |
| **03 · Фильтр** | Прогоняем каждое столкновение через реальные ограничения: стек, темп, рынок, бюджет. Что не собрать — выбраковываем или переформулируем. |
| **04 · Детектор** | На выходе не «вектор», а конкретный собираемый MVP: что строим, для кого, почему работает стык, первый шаг. |

### Формат вывода

Каждая идея — карточка столкновения:

```
⚡ [нить A] ✕ [нить B]
Название: краткое, запоминающееся (3–6 слов)
Идея: что строим и для кого, почему именно стык даёт новизну — под стек и рынок пользователя.
Первый шаг: одно действие, с которого начать сборку.
```

## Установка

Idea Collider — это skill для Claude.ai. Файл скилла — [`idea-collider.skill`](idea-collider.skill), исходник метода — [`SKILL.md`](SKILL.md).

1. Открой Claude.ai → Settings → Capabilities → Skills.
2. Загрузи `idea-collider.skill`.
3. Позови коллайдер: «придумай идею под меня», «коллайдер идей», «столкни мои нити», `collide my ideas`.

Память подгружается в рантайме автоматически — на твоём собственном корпусе.

---

## Честное ограничение

Сила метода — в живой памяти, а не в файле. Сам по себе скилл ничью память не переносит: память и контекст подгружаются в рантайме у того, кто скилл запускает, внутри его сессии. Поэтому коллайдер работает у каждого члена команды на **его собственных** нитях. Передаётся метод и протокол — искра проскакивает только когда коллайдер запущен на реальном корпусе пользователя.

Это ограничение — оно же фича: метод переносим, а искра зажигается на памяти каждого.
