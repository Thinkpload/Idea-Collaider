## § Идеи развития (средний горизонт)

Что делать после launch-чеклиста, не строя бэкенд.

### Скилл
- ~~**v0.2**~~ — сделано: типы стыка (метафора/механика/аудитория/канал/модель + лимит на монокультуру), «второй такт» (копай / мутируй / тройное / ещё без повторов), молчаливый чек-лист перед выдачей.
- **EN-версия `SKILL.md`** — description уже триггерится на английские фразы, но тело метода на русском; английская версия открывает основную аудиторию Claude.ai skills.
- **Пресеты детектора.** Сейчас выход всегда «MVP». Добавить режимы: `контент` (заголовок + тезис + формат), `фича` (в существующий продукт), `эксперимент выходного дня`. Один параметр в запуске, тот же протокол.
- **Парный коллайдер.** Два человека дают по 5 нитей — сталкиваются нити *разных* людей. Формат для команд/кофаундеров; дистанция между людьми обычно больше, чем внутри одной головы.
- **Журнал распадов.** В конце сессии предлагать сохранить выжившие карточки (артефакт/файл), а при следующем запуске «ещё» — читать журнал и не повторяться между сессиями, а не только внутри одной.

- **Теги типа стыка** на демо-карточках (`метафора`, `механика`…) — синхронизация с форматом скилла.

## § Product roadmap — настоящий бэкенд (дальний горизонт)

На лендинге описан полноценный пайплайн «память → профиль → столкновение». Сейчас это **видение**, а не работающий код. Ниже — что нужно построить, чтобы коллайдер работал не только на живой памяти Claude.ai, но и как самостоятельный продукт с явным профилем пользователя.

> Альтернатива всему разделу: остаться **чистым skill** (как сейчас) и не строить бэкенд вообще — память Claude.ai делает всю работу в рантайме. Бэкенд нужен только если хочется явный профиль, кросс-сессионную память и коннекторы вне Claude.ai. **Решить это до того, как вкладываться в инфраструктуру.**

### 1. Онбординг-пайплайн
- Two paths: подключить коннекторы (авто-профиль) **или** вставить нити вручную за 5 минут.
- Настойчиво просить **не-рабочие** нити (хобби, кино, книги) — они дают семантическую дистанцию.
- Показать вытащенный инвентарь обратно для подтверждения и правки.
- Первое столкновение как момент «ага» сразу после онбординга.

### 2. Коннекторы (источники нитей)
Granola · Google Drive · Bitrix24 · Slack · ClickUp · ручной ввод.
- Для каждого — OAuth, маппинг в «нити», политика приватности (что храним, что нет).
- Начать с 1–2 коннекторов (например, ручной ввод + Google Drive), не строить все сразу.

### 3. Дистилляция нитей
- Claude API превращает сырьё в короткие labeled-нити.
- Тег канала: `работа` / `личное`, `тех` / `культура`.

### 4. Эмбеддинги и семантическая дистанция
- Эмбеддинг каждой нити (Vertex / Voyage).
- Дистанция в векторном пространстве = ядро отбора пар (далёкие, но соединимые).
- Калибровка коридора: не слишком близко (банально), не слишком далеко (абсурд).

### 5. Хранилище профиля
- Профиль как JSON + векторный индекс.
- Cloudflare R2 для blob'ов, индекс — векторная БД (Vectorize / pgvector / аналог).
- Приватность: данные пользователя изолированы, удаление по запросу.

### 6. Рантайм-инъекция
- Сборка контекста столкновения из профиля на каждый запуск.
- Связка с skill-протоколом: профиль подаётся как контекст в шаги 01–04.

### Порядок и риски
1. Сначала решить **skill-only vs продукт с бэкендом** (см. врезку выше).
2. Если продукт — начать с ручного ввода + эмбеддинги + одно столкновение end-to-end. Не строить все коннекторы вперёд.
3. Главный риск: инфраструктура (R2, векторный индекс, OAuth коннекторов) — это месяцы работы ради того, что память Claude.ai уже частично делает бесплатно. Подтвердить спрос до вложений.
